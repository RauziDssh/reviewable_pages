# GitHub Issue連携レビュー機能のテスト

このページは、GitHub Pagesでホストされるドキュメントに対して、読者が直接「クリックして指摘（Issue作成）」ができる機能をデモするものです。

### 動作確認の手順
1. 下記のテキストやリスト項目をクリックしてください。
2. ポップアップが表示されるので、指摘内容（例：「誤字があります」など）を入力します。
3. 「Issueを作成」ボタンを押すと、GitHubの新規Issue作成画面が開き、該当箇所のソースコードへのリンクと引用が自動で入力されます。

### サンプルテキスト
Jekyllのカスタムプラグインにより、Markdownから生成されたHTMLの各要素に自動的に行番号が振られています。

- **リスト項目 1**: これをクリックしてもレビューできます。
- **リスト項目 2**: GitHub上でのコードスニペット展開に対応しています。

## 注意事項
GitHub Actionsを使用してビルドを行うため、リポジトリの設定で `Settings > Pages > Build and deployment > Source` を **GitHub Actions** に設定する必要があります。

また、`_config.yml` の `repository_url` を実際のリポジトリURLに書き換えてからプッシュしてください。
